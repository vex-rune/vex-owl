# 依赖管理规范

## 适用范围
Vex-Owl项目所有Maven模块



## 1. 版本管理

### 1.1 集中管理原则
- **所有依赖版本在根pom.xml的properties中定义一次**
- 子模块禁止单独声明version
- 版本通过parent继承获得

### 1.2 正确示例

```xml
<!-- 根pom.xml -->
<properties>
    <java.version>21</java.version>
    <spring-boot.version>3.2.0</spring-boot.version>
    <lombok.version>1.18.30</lombok.version>
</properties>

<!-- 子模块pom.xml -->
<dependencies>
    <dependency>
        <groupId>org.springframework.boot</groupId>
        <artifactId>spring-boot-starter</artifactId>
        <version>${spring-boot.version}</version>  <!-- ❌ 禁止在子模块中定义version -->
    </dependency>
</dependencies>
```

### 1.3 正确做法

```xml
<!-- 子模块pom.xml -->
<dependencies>
    <dependency>
        <groupId>org.springframework.boot</groupId>
        <artifactId>spring-boot-starter</artifactId>
        <!-- 不写version，从parent继承 -->
    </dependency>
</dependencies>
```



## 2. dependencyManagement使用

### 2.1 规范要求
- 根pom.xml使用dependencyManagement统一管理版本
- 子模块dependencies中不指定version
- 特殊版本需要时在子模块中覆盖

### 2.2 依赖原则
- 优先使用Spring Boot管理的依赖版本
- 避免引入与Spring Boot 3.x不兼容的包
- 禁止引入来源不明的依赖



## 3. 禁止规则

| 禁止项 | 说明 |
|-------|------|
| 硬编码版本 | 子模块不能写死版本号 |
| 未知来源 | 禁止引入来路不明的包 |
| 已知漏洞 | 禁止引入有安全漏洞的包 |
| 版本冲突 | 避免引入与现有版本冲突的包 |
