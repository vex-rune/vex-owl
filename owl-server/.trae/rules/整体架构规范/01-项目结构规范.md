# 项目结构规范

## 适用范围
Vex-Owl微服务项目所有模块



## 1. 模块结构

### 1.1 顶级模块划分
- **vex-comm**：公共模块，供所有业务模块使用
- **vex-owl-server**：业务主模块

### 1.2 模块规则
- 业务模块必须继承 vex-owl-server/pom.xml
- 公共模块必须继承 vex-comm/pom.xml
- **禁止新增其他顶级模块**



## 2. 源码目录

```
项目根目录/
├── src/
│   ├── main/
│   │   ├── java/          # Java源码
│   │   └── resources/     # 配置文件
│   └── test/
│       ├── java/          # 测试源码
│       └── resources/     # 测试配置
```



## 3. DDD分层结构

### 3.1 标准结构

```
com.vex.owl.{模块名}
├── api                    # 接口层
│   ├── XXXApi.java       # 接口控制器
│   ├── request            # 请求参数DTO
│   │   └── XxxRequest.java
│   └── response           # 响应DTO
│       └── XxxResponse.java
├── app                    # 应用层
│   ├── XXXApp.java       # 应用服务
│   └── manager           # 应用管理器
│       └── XxxManager.java
└── domain                 # 领域层
    ├── {领域1}/           # 领域子包
    │   ├── entity/
    │   ├── service/
    │   ├── repository/
    │   └── exception/
    └── {领域2}/
        └── ...
```

### 3.2 各层职责

| 层级 | 职责 | 约束 |
|------|------|------|
| **api** | HTTP接入、参数校验、响应封装 | 仅依赖 app 层 |
| **app** | 业务流程编排、事务控制、跨领域协作 | 仅依赖 domain 层 |
| **domain** | 核心业务逻辑、领域模型、数据管理 | 可使用Spring基础能力，构建Manager管理类 |

### 3.3 领域分包规范

每个领域必须包含四个标准组件：

```
{领域名}/
├── entity/                # 领域实体
│   └── Xxx.java          # 核心业务对象
├── service/              # 领域服务
│   └── XxxService.java   # 领域业务逻辑
├── repository/           # 仓储接口
│   └── XxxRepository.java # 数据访问抽象
└── exception/            # 领域异常
    └── XxxException.java # 领域特定异常
```



## 4. 包名规范

| 类型 | 格式 | 示例 |
|-----|------|------|
| 根包 | com.vex | - |
| 业务包 | com.vex.owl.{模块名} | com.vex.owl.user |
| 公共包 | com.vex.{组件名} | com.vex.common |
| 领域包 | com.vex.owl.{模块名}.domain.{领域名} | com.vex.owl.user.domain.subject |
