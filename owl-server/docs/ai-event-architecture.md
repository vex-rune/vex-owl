

# AI 事件系统架构文档

> 本文档描述 AI 事件系统的完整架构，包括事件串联、单机/分布式实现、统计记录和入口限制。

### 4.2 单机版组件

```
┌─────────────────────────────────────────────────────────────────┐
│                      LocalEventBus                              │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   @Component                                                    │
│   public class LocalEventBus implements EventBus {             │
│                                                                 │
│       private final ApplicationEventPublisher publisher;       │
│       private final EventStore eventStore;                     │
│       private final UsageStatistics usageStats;                │
│                                                                 │
│       @Override                                                 │
│       public void publish(Event event) {                       │
│           // 1. 记录到存储                                       │
│           eventStore.save(event);                               │
│                                                                 │
│           // 2. 发布到 Spring Event                              │
│           publisher.publishEvent(toSpringEvent(event));         │
│                                                                 │
│           // 3. 更新统计                                         │
│           if (event instanceof TokenUsageEvent) {               │
│               usageStats.incrementToken(                       │
│                   event.getTenantId(),                          │
│                   ((TokenUsageEvent) event).getTotal()           │
│               );                                                │
│           }                                                      │
│       }                                                        │
│   }                                                             │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 5.3 分布式一致性

```
┌─────────────────────────────────────────────────────────────────┐
│                      分布式一致性策略                            │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   ┌─────────────────────────────────────────────────────────┐  │
│   │                   事件顺序                                │  │
│   ├─────────────────────────────────────────────────────────┤  │
│   │                                                                 │  │
│   │   同一 traceId 的事件保证顺序                              │  │
│   │                                                                 │  │
│   │   • Kafka: partition key = traceId                       │  │
│   │   • Redis: 使用 Redis Stream 保证顺序                      │  │
│   │                                                                 │  │
│   └─────────────────────────────────────────────────────────┘  │
│                                                                 │
│   ┌─────────────────────────────────────────────────────────┐  │
│   │                   事件幂等                                │  │
│   ├─────────────────────────────────────────────────────────┤  │
│   │                                                                 │  │
│   │   事件 ID = traceId + eventType + timestamp              │  │
│   │                                                                 │  │
│   │   消费前检查:                                              │  │
│   │   if (processedEvents.contains(eventId)) {                │  │
│   │       return; // 跳过                                      │  │
│   │   }                                                        │  │
│   │   processedEvents.add(eventId);                           │  │
│   │                                                                 │  │
│   └─────────────────────────────────────────────────────────┘  │
│                                                                 │
│   ┌─────────────────────────────────────────────────────────┐  │
│   │                   事件存储                                │  │
│   ├─────────────────────────────────────────────────────────┤  │
│   │                                                                 │  │
│   │   • 本地处理: 立即存储到数据库                             │  │
│   │   • 消费处理: 批量写入，提高吞吐量                        │  │
│   │                                                                 │  │
```
