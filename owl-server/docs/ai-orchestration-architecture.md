## 二、核心设计原则

### 2.1 分层架构

```
┌────────────────────────────────────────────────────┐
│                   接入层 (ChatApp)                 │
│            职责：路由、校验、响应封装               │
├────────────────────────────────────────────────────┤
│                   编排层 (Orchestrator)            │
│            职责：流程控制、状态管理、事件发布       │
├────────────────────────────────────────────────────┤
│                   能力层                           │
│  ┌─────────────┐ ┌─────────────┐ ┌─────────────┐   │
│  │ SkillExecutor│ │ ChatManager │ │ ToolServer │   │
│  │  技能执行   │ │  会话管理   │ │  工具管理   │   │
│  └─────────────┘ └─────────────┘ └─────────────┘   │
├────────────────────────────────────────────────────┤
│                   基础设施层                       │
│  ┌─────────────┐ ┌─────────────┐ ┌─────────────┐   │
│  │ LLM Factory │ │ ChatMemory  │ │EventPublisher│  │
│  │  模型工厂   │ │  对话记忆   │ │  事件发布   │   │
│  └─────────────┘ └─────────────┘ └─────────────┘   │
└────────────────────────────────────────────────────┘
```

### 2.2 消息驱动

- Agent 之间通过消息通信，不直接调用
- 消息自包含：发送者、接收者、内容、时间戳、元数据
- 消息可追溯：全链路追踪和调试

### 2.3 工具绑定

- 专业 Agent 绑定专属工具集
- 工具隔离，避免权限混乱
- 按需分配，根据任务类型选择对应 Agent

## 四、Pipeline 编排模式

### 4.1 顺序管道 (SequentialPipeline)

```
┌─────────────────────────────────────────────────────────────────┐
│                    SequentialPipeline                           │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   输入 ──▶ Agent1 ──▶ 输出1 ──▶ Agent2 ──▶ 输出2 ──▶ ... ──▶ 结果
│                                                                 │
│   典型场景：                                                      │
│   • 内容创作：选题 → 大纲 → 正文 → 润色                          │
│   • 代码开发：需求分析 → 设计 → 编码 → 测试                      │
│   • 数据处理：采集 → 清洗 → 分析 → 可视化                        │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```
