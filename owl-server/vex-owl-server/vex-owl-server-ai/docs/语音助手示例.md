# 语音助手示例

基于Windows系统的语音识别和语音合成，结合DeepSeek AI实现的语音交互示例。

## 功能特性

- 🎤 **语音识别**: 使用Windows Speech Recognition捕获麦克风输入
- 🔊 **语音合成**: 使用Windows TTS引擎输出语音
- 🤖 **AI对话**: 调用DeepSeek API进行智能问答
- 💬 **流式响应**: 支持打字机效果的流式输出

## 使用前提

### 1. Windows语音功能

确保Windows系统启用了语音识别功能：
- 设置 → 隐私 → 语音 → 启用"在线语音识别"
- 确保麦克风正常工作

### 2. 配置API密钥

在 `application.yml` 中配置DeepSeek API密钥：

```yaml
spring:
  ai:
    deepseek:
      api-key: your-deepseek-api-key
```

### 3. Maven依赖

本模块已包含以下依赖：
- `spring-ai-deepseek`: DeepSeek模型支持
- `spring-boot-starter-webflux`: WebClient支持

## 技术实现

### 1. 语音识别 (STT)

使用PowerShell调用Windows Speech Recognition API：

```
用户说话 → Windows Speech Recognition → 文字
```

关键代码：
- `WindowsSpeechRecognition.java`: 核心识别逻辑
- 使用 `System.Speech.Recognition.SpeechRecognitionEngine`
- 超时控制: 10秒（可配置）

### 2. 语音合成 (TTS)

使用PowerShell调用Windows TTS引擎：

```
文字 → Windows TTS → 语音播放
```

关键代码：
- `WindowsTextToSpeech.java`: 核心合成逻辑
- 使用 `System.Speech.Synthesis.SpeechSynthesizer`
- 支持选择语音（如 "Microsoft Yaoyao"）

### 3. DeepSeek集成

使用Spring WebClient调用DeepSeek Chat API：

```
文字 → DeepSeek API → AI回复
```

关键代码：
- `DeepSeekClient.java`: API调用封装
- 支持流式输出和普通模式
- 错误处理和重试机制

### 4. 主控制逻辑

整合所有组件的协调器：

```
语音输入 → AI处理 → 语音输出 → 循环
```

关键代码：
- `VoiceAssistant.java`: 主交互逻辑
- 支持打断和退出
- 异常恢复机制

## 注意事项

### ⚠️ Windows语音限制

- 语音识别需要网络连接（在线模式）
- 首次使用可能需要在系统设置中授权
- 识别准确度依赖麦克风质量和环境噪音

### ⚠️ 性能考虑

- 语音识别有10秒超时限制
- API调用可能有网络延迟
- 建议在安静环境使用

### ⚠️ 错误处理

- 网络异常：会自动重试并提示用户
- 超时：会提示"没有听到您的声音"
- API错误：会播报错误信息

## 常见问题

### Q: 语音识别不工作
**A**: 
1. 检查Windows隐私设置中是否启用了语音识别
2. 确保麦克风被识别为默认音频设备
3. 在安静环境中测试

### Q: TTS声音奇怪
**A**: 
1. 更改语音名称，如 "Microsoft Yaoyao", "Microsoft Huihui"
2. 在Windows声音设置中调整语音属性
3. 检查系统语言设置

### Q: API调用失败
**A**: 
1. 验证API密钥是否有效
2. 检查网络连接
3. 确认账户余额（DeepSeek按调用量计费）

### Q: 识别速度慢
**A**: 
1. 减少超时时间（如5秒）
2. 在安静环境使用
3. 使用高质量麦克风
