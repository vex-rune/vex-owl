# 通用基础规范

## 适用范围
所有项目、所有团队成员、所有模块



## 1. 分支管理规范

### 1.1 分支命名
| 分支类型 | 命名格式 | 示例 |
|---------|---------|------|
| 主分支 | main | main |
| 开发主分支 | develop | develop |
| 功能分支 | feature/{功能名} | feature/user-auth |
| 修复分支 | fix/{问题描述} | fix/login-bug |
| 发布分支 | release/v{版本号} | release/v1.0.0 |

### 1.2 分支规则
- **main**：线上稳定版，只接受合并，禁止直接提交
- **develop**：开发主分支，所有功能合并到此分支
- **feature**：从 develop 创建，完成后合并回 develop
- **fix**：从 main 或 develop 创建，修复后合并回源分支
- **release**：发布准备分支，测试完成后合并到 main 和 develop



## 2. 提交规范

### 2.1 提交信息格式
```
类型: 描述
```

### 2.2 类型标识
| 类型 | 说明 |
|-----|------|
| feat | 新功能 |
| fix | Bug修复 |
| docs | 文档更新 |
| refactor | 代码重构 |
| test | 测试相关 |
| chore | 构建/工具变更 |

### 2.3 示例
```
feat: 添加用户登录功能
fix: 修复登录超时问题
docs: 更新API文档
refactor: 重构用户服务层
```

### 2.4 禁止提交
- 本地配置文件
- 敏感信息（密码、密钥、token）
- 临时文件和调试代码
- 编译产物



## 3. 代码审查规范

### 3.1 审查标准
- 代码是否符合项目规范
- 业务逻辑是否正确
- 是否存在安全隐患
- 是否有单元测试覆盖
- 注释是否完整清晰

### 3.2 通过条件
- 至少1人审批通过
- 无阻塞性问题
- CI检查全部通过



## 4. 安全规范

### 4.1 绝对禁止
- 硬编码密码、密钥、token
- 绕过权限校验
- SQL注入/XSS漏洞
- 敏感数据明文日志

### 4.2 必须执行
- 接口权限校验
- 输入参数校验
- 敏感数据过滤
