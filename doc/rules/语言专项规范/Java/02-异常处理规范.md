# Java 异常处理规范

## 适用范围
所有Java模块、所有Java开发人员



## 1. 异常分类

### 1.1 领域异常
- 位置：domain/{module}/exception/
- 命名：XxxException.java
- 父类：RuntimeException
- 用途：领域层业务校验失败

### 1.2 应用异常
- 位置：app层或统一异常处理
- 命名：BusinessException.java
- 用途：业务流程异常

### 1.3 系统异常
- 位置：统一异常处理
- 命名：SystemException.java
- 用途：系统级错误（数据库、IO等）



## 2. 异常处理原则（文字规格）

### 2.1 禁止行为
- **禁止空catch块**：catch后必须有处理逻辑
- **禁止捕获不处理**：不能catch后只写pass或空方法
- **禁止吞掉异常**：不能catch后不记录也不抛出
- **禁止捕获Exception**：应捕获具体异常类型

### 2.2 正确做法
- 捕获具体异常类型
- 记录完整日志（包含堆栈）
- 抛出明确的业务异常或直接向上传播
- 给用户返回明确的错误提示

### 2.3 日志要求
- 记录异常消息
- 记录堆栈信息（使用log.error的三个参数形式）
- 记录相关上下文信息（订单号、用户ID等非敏感数据）

### 2.4 异常链
- 保留原始异常信息
- 使用有意义的业务消息包装
- 避免丢失异常原因



## 3. 全局异常处理

### 3.1 统一异常处理器
- 使用Spring的@ControllerAdvice或@RestControllerAdvice
- 针对不同异常类型返回统一格式
- 日志记录所有异常

### 3.2 返回格式
```json
{
    "code": "ERROR_CODE",
    "message": "用户可见的错误信息",
    "data": null
}
```



## 4. 代码规范（纯示例）

### 4.1 领域异常定义

```java
package com.vex.owl.user.domain.subject.exception;

import lombok.Getter;

/**
 * 用户主体异常
 */
@Getter
public class SubjectException extends RuntimeException {
    
    private static final String PREFIX = "USER_";
    
    public static SubjectException notFound() {
        return new SubjectException(PREFIX + "001", "用户不存在");
    }
    
    public static SubjectException invalidStatus() {
        return new SubjectException(PREFIX + "002", "用户状态无效");
    }
    
    private final String code;
    
    public SubjectException(String code, String message) {
        super(message);
        this.code = code;
    }
    
    public SubjectException(String message) {
        super(message);
        this.code = null;
    }
}
```

### 4.2 异常处理示例（正确）

```java
package com.vex.owl.user.app;

import lombok.RequiredArgsConstructor;
import lombok.extern.slf4j.Slf4j;
import org.springframework.stereotype.Service;

@Slf4j
@Service
@RequiredArgsConstructor
public class UserService {
    
    private final SubjectRepository subjectRepository;
    
    public Subject getUser(String id) {
        try {
            Subject subject = subjectRepository.findById(id);
            if (subject == null) {
                throw SubjectException.notFound();
            }
            return subject;
        } catch (SubjectException e) {
            throw e;
        } catch (Exception e) {
            log.error("查询用户失败, id={}", id, e);
            throw new BusinessException("查询用户失败");
        }
    }
}
```

### 4.3 异常处理示例（禁止）

```java
// ❌ 禁止：空catch块
try {
    doSomething();
} catch (Exception e) {
    // 什么都不做
}

// ❌ 禁止：捕获不处理
try {
    doSomething();
} catch (Exception e) {
    log.info("发生异常");
}

// ❌ 禁止：捕获Exception太宽泛
try {
    doSomething();
} catch (Exception e) {  // 应该捕获具体异常
    log.error("错误", e);
}
```

### 4.4 全局异常处理示例

```java
package com.vex.owl.user.api;

import lombok.extern.slf4j.Slf4j;
import org.springframework.web.bind.annotation.ExceptionHandler;
import org.springframework.web.bind.annotation.RestControllerAdvice;

@Slf4j
@RestControllerAdvice
public class GlobalExceptionHandler {
    
    @ExceptionHandler(SubjectException.class)
    public ApiResponse<Void> handleSubjectException(SubjectException e) {
        log.warn("业务异常: {}", e.getMessage());
        return ApiResponse.error(e.getMessage());
    }
    
    @ExceptionHandler(BusinessException.class)
    public ApiResponse<Void> handleBusinessException(BusinessException e) {
        log.error("业务异常", e);
        return ApiResponse.error(e.getMessage());
    }
    
    @ExceptionHandler(Exception.class)
    public ApiResponse<Void> handleException(Exception e) {
        log.error("系统异常", e);
        return ApiResponse.error("系统错误，请稍后重试");
    }
}
```
