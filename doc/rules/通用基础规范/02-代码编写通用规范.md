# 代码编写通用规范

## 适用范围
所有项目、所有语言、所有模块



## 1. 注释规范

### 1.1 必须注释的内容
- 所有类定义
- 公共方法（public）
- 复杂业务逻辑
- 重要算法逻辑

### 1.2 注释原则
- 简洁明了，让新手能快速理解
- 避免过度注释
- 代码本身要自解释
- 行内注释说明"为什么"而非"是什么"



## 2. 日志规范

### 2.1 日志级别使用
| 级别 | 使用场景 |
|-----|---------|
| DEBUG | 开发调试信息 |
| INFO | 正常业务流程节点 |
| WARN | 潜在问题但不影响运行 |
| ERROR | 错误异常信息 |

### 2.2 绝对禁止
- 日志输出密码、密钥
- 日志输出手机号、身份证等隐私信息
- 日志输出完整银行账号

### 2.3 必要日志
- 接口入口/出口
- 业务关键节点
- 异常发生位置
- 耗时操作（超过阈值）



## 3. 异常处理规范

### 3.1 禁止行为
- 空catch块
- 捕获异常不处理
- 吞掉异常信息

### 3.2 正确做法
```java
try {
    // 业务逻辑
} catch (SpecificException e) {
    log.error("操作失败: {}", e.getMessage(), e);
    throw new BusinessException("明确的用户提示");
}
```

### 3.3 统一异常处理
- 使用全局异常处理器
- 统一返回错误码和消息
- 记录完整堆栈信息



## 4. 数据校验规范

### 4.1 入参校验
- 所有接口入参必须校验
- 使用框架提供的校验注解
- 校验失败返回明确错误信息

### 4.2 防注入
- SQL参数使用预编译
- XSS输入过滤或转义
